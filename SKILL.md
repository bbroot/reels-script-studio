# Reels Script Studio（15秒爆款短视频脚本生成器）

生成一份**可直接用于即梦/小云雀/Seedance/剪映**的专业 15 秒短视频脚本：分镜表 + 平台兼容 AI 提示词
+ 设备标注 + 质量自检，并附「用户使用该成品的说明」。支持选项式交互与自进化。

## 一、触发条件

用户说出以下任意一句（或语义等价）时立即触发：

- "帮我写一个15秒短视频脚本" / "生成一个 Seedance/即梦/小云雀的脚本"
- "我要拍产品种草视频" / "给我一个机械臂运镜的脚本方案"
- "生成爆款脚本" / "写个带货短视频脚本" / "AI 视频提示词怎么写"

## 二、第一轮：展示选项菜单（用户做选择题）

触发后，**不要**直接写脚本，先输出以下 5 维选项菜单，并提示回复格式 `1A,2E,3A,4C,5B`：

```
🎬 欢迎使用【15秒爆款脚本生成器】v3.0
请按格式回复，例如：1A,2B,3C,4D,5B

1️⃣ 视频类型      A.产品种草  B.知识科普  C.剧情叙事  D.Vlog生活  E.测评对比  F.热点评论
2️⃣ 专业运镜      A.机械臂环绕  B.子弹时间  C.推拉摇移  D.微距探针  E.综合型
3️⃣ 光影风格      A.好莱坞布光  B.赛博朋克  C.自然柔光  D.高对比硬光  E.极简白
4️⃣ 目标平台      A.即梦  B.小云雀  C.Seedance  D.剪映  E.多平台通用
5️⃣ 核心卖点数量  A.1个超级卖点  B.2个核心卖点  C.3个分层卖点  D.情感共鸣  E.自定义
```

解析示例 `1A,2E,3A,4C,5B` → 产品种草 / 综合型 / 好莱坞布光 / Seedance / 2个卖点。

## 三、第二轮：补充信息收集

收到选项后，主动询问（必填项加 ⭐）：

```
已收到你的选择（1A,2E,3A,4C,5B）。为生成更精准脚本，请补充：
⭐ 产品/主题名称：______
⭐ 核心卖点（按选择数量）：①______ ②______（③______）
选填（越详细越准）：目标人群/痛点、参考风格或对标账号、现成素材
特殊需求：是否同时输出【即梦/小云雀/Seedance 兼容提示词】？（默认是）
          是否输出【剪映 XML 格式】？（默认否）
```

收集完成后进入生成。若用户首轮就提供了全部信息，可跳过菜单直接生成。

## 四、生成：15 秒标准分镜结构

严格按以下时间轴分配（时长可调，需在开头声明）：

| 时段 | 段落 | 任务 |
|------|------|------|
| 0–2s | 钩子 Hook | 反常识/痛点/悬念，3 秒内必须留住人 |
| 2–5s | 高光展示 | 子弹时间/机械臂特写，制造记忆点 |
| 5–9s | 功能演示 | 卖点落地，画面+台词印证 |
| 9–12s | 效果对比 | 前后/竞品对比，强化价值 |
| 12–15s | CTA | 行动指令：点赞/下单/关注，明确锚点 |

### 输出格式（Markdown）

每次输出都包含以下区块，顺序固定：

```markdown
# 🎬 {产品名} · 15秒爆款脚本（{类型} / {平台}）

> 元数据：Skill v3.0.0 · 进化 v{X} · 生成时间 {YYYY-MM-DD HH:MM} · 选择 1A,2E,3A,4C,5B · 哈希 {8位}

## 分镜表
| 镜号 | 时间 | 景别 | 画面描述 | 台词/旁白 | 音效 | 运镜/设备 |
|------|------|------|----------|-----------|------|-----------|
| 1 | 0–2s | 特写 | ... | ... | ... | 机械臂 45° 下推 |

## AI 生成提示词（{平台} 直复制）
主体 + 运镜 + 光照 + 风格 四段式；含专业术语（如 Crane up, Dolly in）。

## 平台适配参数
分辨率/比例/模型模式建议（见 references/platform-adapters.md）。

## 质量自检
钩子强度 / 人物一致性 / 光影真实度 / 运镜种类≥2 / 听感 250–280 字/分。

## 📖 用户使用该成品的说明
（见 references/user-guide.md 对应章节，按需节选精简版）
```

具体 6 类模板的镜头侧重与台词范式见 `references/templates.md`；
四平台提示词结构与参数见 `references/platform-adapters.md`；
用户使用该成品的完整双语说明见 `references/user-guide.md`。

## 五、质量校验标准（生成时必须自检）

- **钩子强度**：前 3 秒必须有明确钩子（反常识/痛点/悬念），否则主动加强第 1 镜。
- **人物一致性**：提示词含「同一人物，面部一致，无变形」约束。
- **光影真实度**：优先含 `volumetric lighting, global illumination, ray tracing`。
- **专业运镜**：至少 2 种专业运镜（2️⃣ 选 E 综合型则自动满足）。
- **听感流畅**：台词控制在 250–280 字/分钟，避免 AI 配音卡顿。
- **违禁词**：带货脚本规避「第一/最/国家级」等绝对化用语；抖音严于小红书（见 user-guide）。

## 六、自进化机制（evolution.json）

经验沉淀在独立的 `evolution.json`，**不修改核心 SKILL.md**，实现"核心冻结、经验累积"。

- **触发**：用户反馈（"第2镜子弹时间再明显点"）、手动改稿后询问保存、每月频率统计自动优化。
- **记录**：用 `scripts/evolve.py` 写入 `usage_stats` / `user_preferences` / `experience_log`，
  失败修改入 `rejected_optimizations`，防止重复踩坑。
- **回滚**：进化后效果下降 >20% 自动回滚到上一稳定版本。
- **查看**：用户说"报告"时，输出进化报告（使用偏好 + 经验数 + 建议）。

核心文件（`SKILL.md`、模板、运镜库、质量校验）为**冻结层**，禁止在进化中改动结构，仅可调整参数值。

## 七、与其他系统兼容

- 与 "github-to-skills + skill-manager + skill-evolution-manager" 三件套兼容：evolution.json 可直接被读取。
- 遵循 Claude Skill-creator 规范，可用于自动化质量测试与多代理平台适配验证。
